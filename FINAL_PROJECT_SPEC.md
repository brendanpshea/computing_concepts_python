# COMP 1150: Final Programming Project

**Total Points:** 15 points  
**Submission Location:** D2L Assignment Folder  
**Format:** Google Colab Notebook OR Local Python Project (VS Code) + GitHub + Video Demo  

---

## 🎯 Overview

Congratulations on reaching the end of the course! For your final project, you will build a Python program that demonstrates what you've learned. 

You can **take an exercise or capstone project from class and build upon it**, or **create a brand-new Python project** of your own choice. As long as it is written in Python and meets the requirements below, any topic is welcome!

---

## 🛠️ Project Options

Choose **one** of the following development environments:

- **Option A: Google Colab Notebook**  
  Write your code and documentation in a Google Colab notebook (`.ipynb`), saved directly to a public or shared GitHub repository.
- **Option B: Standalone Python Program**  
  Build a script or multi-file project locally using **VS Code** (or your favorite IDE) and push your files to a GitHub repository.

---

## 💡 Project Ideas & Inspiration

Need ideas? Here are a few ways to extend what we covered in class:
- **Interactive Story or Game:** Expand a text adventure with inventory management, stats, or branching paths.
- **Data & Collections Manager:** Create a program to track a collection (movies, game stats, study log) using lists of dictionaries or SQLite.
- **Utility Tool:** Build a password strength checker, Caesar cipher encoder/decoder, or unit converter.
- **Class-Based Simulation:** Use Object-Oriented Programming (OOP) to model something (e.g., a virtual pet, bank account system, or dice game).

---

## 📋 Project Requirements & Grading (15 Points Total)

| Requirement | Description | Points |
| :--- | :--- | :---: |
| **1. Working Python Code** | Code runs smoothly without syntax errors. Uses foundational Python concepts correctly (variables, loops, conditionals, functions, collections, or classes). | **5 pts** |
| **2. Code Explanation & Readme** | Clear code comments/docstrings explaining how functions work. Includes a markdown header/README summarizing what the program does and how to run it. | **3 pts** |
| **3. AI Reflection Statement** | You are **allowed to use AI tools** (ChatGPT, Gemini, Copilot, etc.)! You must include a short paragraph (3–5 sentences) in your reflection describing how you used AI (e.g., what prompts you used, what code it helped draft, and how you verified or fixed it). | **2 pts** |
| **4. Video Demo** | A short (2–3 minute) screen recording demonstrating your program in action and giving a brief walkthrough of your code. | **3 pts** |
| **5. GitHub & Submission** | Code is uploaded to a GitHub repository. Submitted on time with all required links. | **2 pts** |

## 📹 Video Demo Instructions

Record a short **2 to 3 minute** video showing:
1. **Demonstration:** Run your program and show its features working.
2. **Code Walkthrough:** Briefly highlight 1 or 2 interesting functions or logic choices in your code.
3. **Format:** Upload your video to **YouTube (set to Unlisted)** or **MediaSpace**, and copy the share link.

## 📥 How to Submit (D2L Folder)

Submit the following **3 items** to the D2L assignment folder:

1. 🔗 **GitHub Repository Link** (pointing to your project repo).
2. 📄 **Copy of your code file** (your `.ipynb` notebook file or `.py` file).
3. 🎥 **Link to your Video Demo** (YouTube unlisted or MediaSpace URL).
